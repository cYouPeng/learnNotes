进入一个文件夹，git init  把这个夹初始化为git仓库，里边会有一个.git文件夹，默认隐藏

git config有三个级别分别是system，global，local，从高到低，低的会覆盖高的相同的配置
一般新安装git的时候需要设置一下
git config --global user.email"you@example.com"

git config --global user.name"你的名字"     这个设置就是为了在提交历史中显示你的名字，加以区分

git config --global/--system/--local -list   可以查看三个级别的设置
直接输入git config 会显示帮助文件


如果你想连接github的话，需要先在本地生成一个秘钥对，执行 ssh-keygen -t rsa -C "youremail@example.com".
然后会在本地生成一个秘钥对，id_rsa为私钥，id_rsa.pub为公钥，本地地址 屏幕会给你显示出来
然后我们把公钥里的内容放到github上，settings -> ssh key -> add key 标题不用管，
为什么要公钥私钥呢，你在推送的时候，github不光验证你的账号，还会拿github上的公钥去和你本地的私钥进行匹配，匹配上才可以推送。
如果你换了一台电脑，虽然是一样的账户，但是由于私钥不一样，也推送不了

然后：在本地添加remote

git remote add origin git@github.com:707296/learnNotes.git
origin：这是你在本地给你连接的远程库起一个名字，后续如果你添加多个remote，那么这个名字是不可以重复的，但可以随便起

假如你起了一个git remote add hello xxxxx   hello就是xxxxx这个远程库在你本地的名字
那么以后就得这样  git pull hello master
